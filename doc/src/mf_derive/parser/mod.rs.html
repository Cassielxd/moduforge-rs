1//! 解析模块
2//!
3//! 提供宏属性解析、字段分析和配置验证的完整功能。
4//! 严格遵循 SOLID 设计原则，确保解析逻辑的清晰性和可维护性。
5//!
6//! # 模块组成
7//!
8//! - `attribute_parser`: 属性解析器，负责解析宏属性并构建配置对象
9//! - `field_analyzer`: 字段分析器，负责分析结构体字段的类型和属性信息
10//! - `validation`: 验证器，负责验证配置的正确性和完整性
11//! - `default_value`: 默认值处理器，负责解析和处理字段默认值
12//!
13//! # 设计原则体现
14//!
15//! - **单一职责原则 (SRP)**: 每个子模块都有明确的单一功能
16//! - **接口隔离原则 (ISP)**: 提供专门的解析、分析和验证接口
17//! - **开闭原则 (OCP)**: 通过配置化支持扩展新的解析规则
18//! - **里氏替换原则 (LSP)**: 所有解析器都可以无缝替换使用
19//!
20//! # 使用流程
21//!
22//! 1. **属性解析**: 使用 `AttributeParser` 解析宏属性
23//! 2. **字段分析**: 使用 `FieldAnalyzer` 分析字段类型和属性
24//! 3. **配置验证**: 使用 `Validator` 验证解析结果的正确性
25//! 4. **错误处理**: 统一的错误类型和友好的错误消息
26//!
27//! # 示例
28//!
29//! ```rust
30//! use syn::parse_quote;
31//! use crate::parser::{AttributeParser, FieldAnalyzer, Validator};
32//!
33//! let input: syn::DeriveInput = parse_quote! {
34//!     #[derive(Node)]
35//!     #[node_type = "paragraph"]
36//!     struct ParagraphNode {
37//!         #[attr]
38//!         content: String,
39//!     }
40//! };
41//!
42//! // 1. 解析属性
43//! let config = AttributeParser::parse_node_attributes(&input)?;
44//!
45//! // 2. 验证配置
46//! Validator::validate_node_config(&config)?;
47//!
48//! // 3. 分析字段（如果需要更详细的分析）
49//! let analyses = FieldAnalyzer::analyze_fields(&get_fields(&input))?;
50//! Validator::validate_field_analyses(&analyses)?;
51//! ```
